I chose the question “Explain how creating and using classes supports Object Oriented programming and S.O.L.I.D. design principles, give specific examples”

Creating and using classes supports Object-Oriented programming by helping to organize data and attributes that can later be used in more complex projects. A class is pretty much like a blueprint in coding, and you can add whatever attributes you want, specify how the constructor should be formed, and it can even help developers with polymorphism and inheritance. A good example is that a Car class can have simple functions like start() and stop(). But if a developer wants to make a specific vehicle make like Tesla, they can make a Tesla class, make it inherit from the parent class Car, and override the methods already stored in Car. This is a great example of how classes are used and created by developers.
Classes also help developers to follow the S.O.L.I.D design principles. It is a simple way to make it easier to maintain, scale, and understand code over time. An in-depth example is the Single Responsibility Principle. This is where each class has only one job, and it is only in charge of that. If a Friends class stores only the names of the developer’s friends and if the FriendsFavFood class stores the favorite food of the friends, they are separate and only focus on what they were created and programmed to do. This makes it more organized and less stressful for the developer.
